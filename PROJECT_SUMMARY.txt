                         CORE FEATURES
================================================================================

FOR ADMINS:
  * One-time admin registration (single admin account)
  * Subject/category management
  * PDF/DOCX file upload with auto-parsing
  * Three answer marking patterns (# at start, ++++ at end, separate file)
  * AI-powered parsing (GPT-4o-mini) with regex fallback
  * Question management and preview
  * Access code generation (with subject restriction & attempt limits)
  * Results tracking with detailed user answers
  * Publish/unpublish tests
  * Test session review

FOR USERS:
  * Simple login with 8-character access code + name (no password)
  * Multiple users per single code
  * Test selection by subject with grid layout
  * Two test modes:

    - Batch-25: 25 questions at once with timer
  * Automatic scoring on submission
  * Detailed result review with answer comparison
  * Pass/fail indicators

TECHNICAL:
  * MathJax 3.x for rendering mathematical formulas
  * Equation extraction from Word documents
  * OpenAI GPT-4o-mini API integration
  * Fallback regex parsing if API unavailable
  * Responsive CSS Grid + Flexbox design
  * Mobile-friendly viewport
  * Session-based authentication
  * Custom admin panel (no Django admin site)

================================================================================
                        DATABASE STRUCTURE
================================================================================

7 CORE MODELS:

1. Subject
   - Test categories/subjects
   - name, description, created_at

2. Test
   - Individual tests with files
   - question_file, answer_file (optional)
   - answer_marking (hash_start, plus_end, separate_file)
   - is_parsed, is_published, time_limit, passing_score
   - Relations: has many Questions

3. Question
   - Individual questions from parsed files
   - text, order, created_at
   - Relations: has many Options

4. Option
   - Multiple choice answer options
   - text, is_correct, order
   - Relations: belongs to Question

5. AccessCode
   - 8-character unique codes (e.g., "AB12CD34")
   - One code for many users
   - Optional: subject restriction, max attempts per user, expiration
   - Methods: is_valid(), get_users()

6. TestSession
   - User test attempts
   - Supports: one_by_one or batch_25 modes
   - Tracks: score, correct_answers, total_questions
   - Multiple users per code
   - Method: calculate_score()

7. UserAnswer
   - Individual answers per question per session
   - Tracks: selected option, correctness
   - Foreign keys: session, question, option

================================================================================
                         DESIGN HIGHLIGHTS
================================================================================

STRENGTHS:
  * AI-powered parsing for high accuracy
  * Regex fallback system for reliability
  * Multi-user per code design for scalability
  * Dual test modes for flexibility
  * Clean custom admin panel
  * Formula support via MathJax
  * Responsive mobile design
  * Good separation of concerns

DESIGN DECISIONS:
  * Custom admin panel → Better UX than Django admin
  * Session-based user auth → No password management
  * AI with regex fallback → Robustness
  * MathJax rendering → Industry standard
  * Two test modes → Flexibility for different use cases
